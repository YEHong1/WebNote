# 第一章 了解HTTP协议



## 1.1 浏览器通过域名访问网页发生了什么

1.  用户在浏览器中输入域名后，浏览器会根据域名去DNS服务器查询它对应的 `ip` 地址
2.  根据这个 `ip`地址向对应的 WEB服务器 发送请求，服务器回传页面内容
3.  浏览器接收并渲染



## 1.2 HTTP简介

HTTP：

超文本传输协议，是一种通信协议。它允许将超文本标记语言（HTML）文档从WEB服务器传送到客户端的浏览器



Web：是一种基于超文本和`HTML`的、全球性的、动态交互的、跨平台的分布式**图形信息系统**。 



http是无状态协议，这保证了它的高效。Cookie 和 Section 的出现使 http 在保持高效的情况下，能够记住状态。



## 1.3 TCP/IP协议

-   HTTP协议是构建在TCP/IP协议上的，是TCP/IP的一个子集。



### TCP/IP协议族

-   TCP/IP协议其实是一系列与互联网相关联的协议集合起来的总称

-   分层管理是TCP/IP协议的重要特征

-   TCP/IP协议族是由一个四层协议组成的系统，这四层分别是 应用层、传输层、网络层和链路层

    ![](.\images\tcp四层结构.png)

    1.  应用层一般是我们编写的应用程序，决定了向用户提供的应用服务。应用层可以系统调用与传输层进行通信，如FTP、DNS、HTTP等
    2.  传输层通过系统调用向应用层提供处于网络连接中的两台计算机之间的数据传输功能。在传输层由两个性质不同的协议：TCP（面向连接） 和 UDP（无连接）。TCP是面向连接的，需要校验，所以更可靠，但效率低， UDP是无连接的，不需要检验，可靠性低，效率更高。
    3.  网络层用来处理在网络上流动的数据包，数据包时网络传输的最小数据单位。该层规定了通过怎样的传输路线到达对方计算机，并把数据包传输给对方。
    4.  链路层用来处理连接网络的硬件部分。硬件上的范畴均在链路层的作用范围之内



### 传输层TCP的三次握手

-   使用TCP协议进行通信的双方必须先建立连接，然后才能开始传输数据。为了确保连接双方的可靠性，TCP协议常用了三次握手策略
    1.  第一次握手：客户端发送请求报文给服务器端，等待服务器端的确认。
    2.  第二次握手：服务端接受到客户端的请求报文（说明客户端的发送能力、和服务端的接受能力正常），向客户端发送响应报文
    3.  第三次握手：客户端接受到服务端的响应报文（说明服务端的发送能力和客户端的接受能力正常），完成TCP的三次握手。



### DNS服务

-   通常我们访问一个网站，使用的时主机名或者域名捞进行访问的。应为相对于IP地址，域名更容易让人记住。但TCP/IP协议使用的是IP地址进行访问的，所以必须有个机制或服务把域名转换为IP地址。DNS服务就是用来解决这个问题的，它提供域名到IP地址之间的解析服务。





## 2.1 HTTP协议的特点

1.  支持客户/服务器模式
    -   客户/服务器模式工作的方式是由客户端向服务器发出请求，服务器端响应请求，并进行相应服务
2.  简单快速
    -   客户端向服务器请求服务时，子需要传送请求方法和路径
    -   请求方法常用的由GET、HEAD、POST。每种方法规定了客户端与服务器联系的类型不同。
    -   由于HTTP协议简单，使得HTTP服务器的程序规模小，因而通信速度很快。
3.  灵活
    -   HTTP允许传输任意类型的数据对象
    -   正在传输的类型由Content-Type加以标记（Content-Type是HTTP包中用来标识内容类型的标识）
4.  无连接
    -   无连接的含义是限制每次连接只处理一个请求
    -   服务器处理客户端的请求，并收到客户端的应答后，即断开连接
    -   采用这种方式可以节省传输时间
5.  无状态
    -   HTTP协议是无状态协议
    -   无状态是指协议对于事务处理没有记忆能力。缺少状态意味着如果后续处理需要前面的信息，则它必须重传，这样可能导致每次连接传送的数据量增大。
    -   另一方面，在服务器不需要先前信息时它的应答就较块



## 2.2 URI与URL

![](.\images\URI.png)

1.  URI可以分为URL和URN， 或者同时具备地址和名称特性的东西。
2.  URN的作用就好像一个人的名字，URL就像这个人的地址
3.  换句话说：URN确定了东西的身份，URL提供了找到它的方式
4.  URL是URI的一种，不是所有的URI都是URL；URI唯一地标识了身份，URL给出了访问机制
